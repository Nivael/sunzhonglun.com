# 英文版翻译手册

本文是 `docs/PRD.md` §7.7 的执行口径。目标不是生产“正确的机器翻译”，而是让英文读者读到
同一个作者：事实忠实、英文自然，仍有 Zhonglun 的呼吸、幽默与判断。

## 内容模型

- `content/posts/` 是唯一中文源稿。
- `content/posts-en/` 放完成的英文译稿，文件名必须与源稿完全相同。
- 英文 frontmatter 只写 `title`、`quote`、`excerpt`；日期、slug、分类、封面从中文源稿取得。
- `quote` 必须逐字出现在英文正文。图片路径沿用源稿，不复制图片。
- 没有译稿就不生成英文页面；不得提交占位稿或只有部分段落的稿件。

## 作者声音

- 叙述通常直接、清醒，不替读者总结感受；情绪越重，语言往往越克制。
- 自嘲是冷面的，笑点不加解释，不主动“写得更好笑”。
- 长句承担记忆与思辨，短句负责转折、落地或突然刺破；不要把所有句子修成同一种长度。
- 口语、俗语与稍显古典的表达可以在同一篇里并存，这是声音的一部分，不统一成单一文体。
- 抒情来自具体物件和动作，不用额外形容词放大：脚、自行车轮、切好的水果、漂亮茶杯都要留下。
- 对死亡、爱、羞耻和自由的判断不软化，也不替作者添加疗愈式结论。

## 四遍工作法

1. **事实对照**：逐段核对人物、时间、动作、因果、否定、数量；段落不得漏译。
2. **声音与节奏**：朗读英文，恢复原文的长短句、重复、停顿、突然转调和冷面笑点。
3. **专名与引文**：核实人名、地名、书名、作品名、互联网语汇；已有权威英文名时采用权威名。
4. **英文润色**：删掉翻译腔和 AI 常见的解释性连接词，但不以“自然”为由改写意义。

完成后再检查：

- 标题是文学标题，不是 SEO 摘要。
- `excerpt` 像 trailer，不剧透；`quote` 与正文逐字一致。
- 图片、强调、引用块、分隔线等 Markdown 结构完整。
- 英文阅读页、中文原文页之间的语言链接双向可达。

## 文化语汇

优先让上下文解释，而不是加脚注。第一次出现且理解情节必需时，可以在句内给一个轻量同位语。
例如“火星文”可写为 `Martian-script screen name`，保留陌生感；“买椟还珠”在引用中直译意象，
因为母亲下一句已经给出含义。不要把中国学校、城市与互联网经验替换成英美类比。

## 新文章流程

新增中文文章时，默认在同一轮工作中：

1. 建立同名英文文件；
2. 完成四遍校订；
3. 从 `tests/translation-backlog.json` 移除该文件（新稿正常不应进入 backlog）；
4. 运行 `npm test` 与 `npm run build`；
5. 在中英文页面互相检查语言切换。

测试会拒绝任何“未翻译、也未登记”的中文稿。若作者明确希望中文先发，可暂不建立译稿，
但须显式登记进 backlog；站点会安全地只发布中文版。
